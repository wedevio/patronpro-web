## Goal

Make the Liverpool Digital website optimizer and Lighthouse analyzer useful for the next mobile LCP pass without live GHL builder, Supabase Storage, or appointment mutation. The pass must reduce generated critical image payload, keep raw Lighthouse evidence, and make budget misses machine-visible.

## Source facts

- Current after-run mobile LCP is 8.2 s.
- Current after-run mobile image transfer is about 700 KiB.
- `logo_square.png` is about 670 KiB and is the dominant current image request.
- The selected mobile hero AVIF is about 29.9 KiB.
- GHL Media rejected AVIF. Supabase public Storage worked for AVIF/WebP/JPEG derivatives.
- Current page-image widths are 640/960/1440.
- Existing analyzer summaries did not retain raw JSON in the repo and did not expose LCP element/phases or budgets.

## Budgets

| Run | Current budget | Target budget |
| --- | --- | --- |
| Mobile performance | >= 70 | >= 90 later |
| Mobile LCP | <= 2500 ms | <= 1000 ms |
| Mobile image transfer | <= 200 KiB | <= 120 KiB later |
| Mobile total transfer | <= 800 KiB | <= 600 KiB later |
| Desktop performance | >= 90 | >= 94 later |
| Desktop LCP | <= 1500 ms | <= 1000 ms |
| Desktop image transfer | <= 800 KiB | <= 500 KiB later |
| Desktop total transfer | <= 1500 KiB | <= 1000 KiB later |

Budget failure must be reportable in JSON and optionally fail the command through `--fail-on-budget`. The sub-second target must remain visible even when current feasibility still fails.

## Implementation notes

### Optimizer

- FILE: `dev/agents/artifacts/script/patronpro-liverpool/optimize-existing-website-html.mjs`
- CURRENT: line 17 uses `WIDTHS = [640, 960, 1440]`.
- PROPOSED: use `WIDTHS = [320, 480, 720, 960, 1440]`, with 320/480/720 treated as mobile candidates and 960/1440 as desktop candidates.
- CURRENT: `<picture>` emits one AVIF source and one WebP source with `sizes="100vw"`.
- PROPOSED: emit media-scoped sources:
  - `(max-width: 767px)` with 320/480/720 candidates and `sizes="100vw"`;
  - `(min-width: 768px)` with 960/1440 candidates and `sizes="100vw"`;
  - JPEG `img` fallback keeps a responsive `srcset`.
- CURRENT: hero preload is one universal AVIF preload with `imagesizes="100vw"`.
- PROPOSED: emit separate mobile and desktop preload links with matching media queries.
- CURRENT: GHL upload attempts every generated page-image format.
- PROPOSED: keep `--apply-upload` and `--supabase-upload` guards, but in GHL mode upload/use JPEG page images only because AVIF is known to fail; Supabase mode remains AVIF/WebP/JPEG.
- CURRENT: `logo_square.png` remains a 670 KiB critical image in live preview.
- PROPOSED: infer the Supabase logo URL from the public image host or accept `--logo-url`, generate 96/192/384 PNG logo candidates, and replace `{{custom_values.logo}}` / `{{custom_values.logo_cuadrado}}` with compact responsive PNG `srcset` values in generated HTML.

- FILE: `dev/agents/artifacts/script/patronpro-liverpool/lighthouse-analyze.mjs`

- LCP element source: `audits["largest-contentful-paint-element"].details.items[0].node.*`.
- LCP phase source: best effort from `audits["largest-contentful-paint"].details.items`, `audits.metrics.details.items[0].lcpResult`, and observed metrics fields. Missing fields must be `null`, empty string, or empty array rather than causing failure.
- Raw Lighthouse JSON generated by `--url` must be written to a new `--out-dir`; do not overwrite older summaries.
